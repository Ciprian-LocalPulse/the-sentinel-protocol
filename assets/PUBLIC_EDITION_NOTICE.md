# Public Edition Notice

## Purpose

This repository is the Public Edition of The Sentinel Protocol.

It exists to provide a procurement-safe, public-facing view of the product concept, brand, governance posture, and evaluation surface.

## Public Edition Includes

The Public Edition may include:

- static landing page
- sanitized product overview
- non-sensitive architecture language
- public preview assets
- ownership and licensing notices
- support and security policies
- contribution rules
- public funding configuration

## Public Edition Excludes

The Public Edition does not include:

- private source logic
- prompts or AI orchestration details
- biometric processing logic
- stylometric model implementation
- negotiation playbooks
- operational automation workflows
- client environments
- API keys, secrets, credentials, or deployment variables
- customer, executive, or communication data
- internal business strategy
- commercial implementation details
- production infrastructure configuration
- restricted private enterprise materials

## Private Enterprise Edition

The Private Enterprise Edition is separate from this public repository.

Private package access, private source review, implementation planning, deployment rights, commercial usage rights, support obligations, exclusivity, and transfer terms are available only through explicit written agreement.

The public repository should not be treated as the complete product, a complete source package, a compliance certification, or a production deployment package.

## Public Discussion Boundary

Public issues and pull requests must remain limited to the public documentation and public preview surface.

Do not request or disclose private source logic, prompts, biometric processing logic, stylometric model details, negotiation playbooks, client environments, credentials, customer data, executive communications, internal strategy, or restricted implementation details in public GitHub channels.

## Procurement-Safe Interpretation

This repository may support initial review, but it is not:

- a security audit
- a privacy certification
- a biometric compliance attestation
- a legal opinion
- a production readiness attestation
- an autonomous execution approval
- a transfer agreement
- an open source release of private materials

Formal diligence, security review, legal review, privacy review, implementation review, or enterprise access review must occur privately under appropriate written terms.
