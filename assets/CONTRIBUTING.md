# Contributing

## Contribution Model

The Sentinel Protocol is maintained as a controlled Public Edition for architecture communication, public documentation, and enterprise evaluation.

Public contributions are reviewed selectively and must stay within the Public Edition boundary.

## Acceptable Public Contributions

Acceptable contribution areas include:

- documentation clarity
- proofreading and structure improvements
- public governance file improvements
- public security policy improvements
- accessibility improvements for public materials
- broken-link reports
- public presentation feedback
- public demo polish that does not alter product behavior

## Not Accepted

Do not submit:

- private source logic
- prompts or AI orchestration details
- biometric processing logic
- stylometric model implementation
- negotiation playbooks
- operational automation workflows
- client environment details
- API keys, secrets, credentials, tokens, or deployment variables
- customer, executive, or communication data
- internal business strategy
- commercial implementation details
- production infrastructure configuration
- material copied from a third party without rights

## Pull Request Requirements

Every pull request must:

- describe the public-facing change clearly
- confirm that no secrets or private material were added
- preserve the Public Edition boundary
- preserve ownership notices
- avoid unsupported security, compliance, privacy, biometric, or financial claims
- avoid changing product logic, code behavior, private architecture, or commercial terms

## Public Issues

Use public issues only for:

- public documentation corrections
- broken public links
- public demo defects
- repository hygiene suggestions
- safe product presentation feedback

Do not use public issues to request private source code, prompts, biometric processing logic, stylometric models, negotiation playbooks, customer data, executive communications, or private enterprise materials.

## Security Reports

Do not report vulnerabilities through public issues or public pull requests.

Use the private reporting process in [SECURITY.md](SECURITY.md).

## Maintainer Discretion

The maintainer may edit, reject, close, or remove submissions that do not align with the Public Edition boundary, legal posture, procurement readiness, privacy expectations, or security expectations of this repository.
